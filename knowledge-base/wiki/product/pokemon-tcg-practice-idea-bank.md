## Summary

This page is a concept bank, not a product spec. The goal is to capture promising directions for a Pokémon TCG practice tool based on patterns that already work elsewhere.

## High-potential idea clusters

### 1. Opening-hand lab

Inspired by Limitless DrawCalc and spaced practice.

Ideas:

- keep / mull decision drills
- opening-hand probability explorer
- "how often do I brick?" deck-health checks
- short daily opening reps instead of full games

### 2. Goldfish sandbox

Inspired by Limitless Tabletop.

Ideas:

- play turn 1–3 lines repeatedly without queue time
- test sequencing in isolation
- rehearse prized-card contingencies or setup routes
- compare two openings from the same list

### 3. Turn quiz / line quiz

Inspired by retrieval practice and Chess.com retry flows.

Ideas:

- pause on a real turn and ask "what’s your line?"
- answer before seeing feedback
- retry the position after review
- build a personal deck of past mistakes as flashcards

### 4. Matchup flashcards

Inspired by deck guides, retrieval practice, and interleaving.

Ideas:

- "What matters in Dragapult vs X?"
- common gust targets
- prize-map reminders
- which support Pokémon to protect / strand / gust

### 5. Post-game retrospective journal

Inspired by Atlassian retrospectives and PokéStats-style coaching.

Ideas:

- 4 Ls adapted to games: loved, loathed, learned, longed for
- sad / mad / glad after-action check-in
- auto-generated action items after a reviewed game
- weekly reflection on recurring leaks instead of isolated losses

### 6. Deck-guide notebook

Inspired by Metafy guide products.

Ideas:

- save matchup notes next to a deck
- collect tech-choice rationales
- attach sample lines and clips/log moments to a deck
- convert static guides into searchable decision notebooks

### 7. Interleaved drill playlists

Inspired by interleaving.

Ideas:

- rotate between matchups and board states in one session
- mix sequencing, prize mapping, and closing turns
- prevent overfitting to one archetype or one repeated puzzle type

### 7b. Daily micro-lessons

Inspired by Chessable and Daily Dose of GTO.

Ideas:

- one five-minute puzzle or concept per day
- spaced resurfacing of old mistakes
- a "you'll be quizzed on this later" memory loop
- short mobile-friendly study instead of only long desktop sessions

### 8. Personal leak tracker

Inspired by GTO Wizard, Untapped, and Mobalytics patterns already captured elsewhere.

Ideas:

- repeated missed KOs
- recurring over-benching
- too-early gust / Iono / recovery patterns
- matchup clusters where confidence drops

### 9. Community practice packs

Inspired by PokemonCard's community layer and Metafy guide market.

Ideas:

- shareable drill sets for a deck or matchup
- creator-authored quiz packs
- "train for this tournament meta" bundles
- coaching homework packs between sessions

### 9b. Tiny companion tools

Inspired by Pokémon Deck Utils and other microtools.

Ideas:

- standalone prize tracker
- opening-hand keeper tool
- matchup note overlay
- stream-friendly training widgets
- one-button deck health checks for bricking/setup patterns

### 10. Faster-than-ladder training modes

This is the broadest theme across the source set.

Ideas:

- opening-only reps
- endgame-only reps
- comeback-from-behind scenarios
- "boss turn" puzzles
- prize-check scenarios

### 11. Tournament prep console

Inspired by Pokedata.

Ideas:

- local event finder tied to a personal prep checklist
- player-history scouting before majors
- compare likely decklists or tech trends before an event
- sanitize / validate decklists for submission workflows

### 12. Deck package lab

Inspired by PokemonCard packages.

Ideas:

- save reusable 4–20 card tech modules
- compare "anti-meta package A vs package B"
- build practice drills around a package rather than a whole deck
- attach matchup notes to a reusable package concept

### 13. Collection-aware practice

Inspired by PokemonCard's collection overlay.

Ideas:

- suggest decks or drills based on cards you already own
- show "owned / needed" alongside practice recommendations
- plan efficient test paths without needing every card immediately

### 14. Custom sandbox formats

Inspired by PokemonCard cubes.

Ideas:

- create custom drill pools
- draft from curated decision environments
- build meta-training rooms with only selected archetypes/cards
- use limited/sandbox modes as a fun way to practice pattern recognition

## Strong meta insight from the source set

Most existing tools help with one of these:

- information lookup
- deck construction
- tournament/meta discovery
- human coaching
- isolated practice utilities

The underexplored space is the system that ties them together into a repeatable improvement loop.

## Idea quality filter

Promising ideas seem to share these traits:

- faster than playing a full live match
- active recall, not passive reading
- grounded in real decks/matchups
- able to create repeat reps over time
- able to surface one next habit instead of generic advice
